# Roadmap: Реализация регистрации и авторизации

Этот документ описывает пошаговый план реализации функционала регистрации и входа в систему (Login & Register) в приложении Poker Planning.

## 1. Состояние системы

### Backend (Python/FastAPI)

- **Готовность: 100%**
- Эндпоинты реализованы: `/auth/register`, `/auth/login`, `/auth/me`.
- Логика хеширования паролей и генерации JWT токенов работает.
- База данных (PostgreSQL) поддерживает модель `User`.

### Frontend (React/TypeScript)

- **Готовность: 20%**
- Есть структура папок в `features/auth` и `entities/user`.
- API-клиент `authApi.ts` содержит заглушки.
- `SessionProvider` имеет заглушку на метод `register`.
- UI-компоненты форм (`LoginForm`, `RegisterForm`) пустые.

---

## 2. План реализации

### Шаг 1: Подготовка типов и API (Shared & Entities)

**Цель:** Обеспечить фронтенд типами данных и методами для взаимодействия с API.

1. **`apps/frontend/src/entities/user/api/authApi.ts`**
   - Реализовать метод `register` (сейчас там заглушка с ошибкой).
   - Убедиться, что метод `login` и `getUser` работают корректно.
2. **`apps/frontend/src/entities/user/model/types.ts`**
   - Уточнить типы `LoginCredentials` и `RegisterCredentials`, чтобы они соответствовали Pydantic-схемам бэкенда.

### Шаг 2: Реализация UI компонентов (Features)

**Цель:** Создать удобные и валидируемые формы.

1. **`apps/frontend/src/features/auth/model/schemas.ts`**
   - Создать схемы валидации Zod для форм (email, password min 8 chars, name).
2. **`apps/frontend/src/features/auth/ui/LoginForm.tsx`**
   - Реализовать форму входа с использованием `react-hook-form` и `@hookform/resolvers/zod`.
   - Использовать общие UI компоненты: `Input`, `Button`.
   - Добавить обработку ошибок API (неверный пароль и т.д.).
3. **`apps/frontend/src/features/auth/ui/RegisterForm.tsx`**
   - Реализовать форму регистрации (Email, Имя, Пароль).
   - Добавить валидацию и обработку ошибок (например, "email уже занят").

### Шаг 3: Обновление управления сессией (App Layer)

**Цель:** Связать формы с глобальным состоянием.

1. **`apps/frontend/src/app/providers/SessionProvider.tsx`**
   - Реализовать метод `handleRegister` (вызов API + сохранение сессии).
   - Убедиться, что после регистрации/входа пользователь перенаправляется на главную страницу или в личный кабинет.

### Шаг 4: Настройка роутинга (App Layer)

**Цель:** Ограничить доступ к страницам.

1. **`apps/frontend/src/app/router/routes.tsx`**
   - Убедиться, что страницы `/login` и `/register` недоступны для уже авторизованных пользователей (редирект на `/`).
   - Настроить защищенные маршруты (например, создание комнаты должно требовать авторизации, если это предусмотрено логикой).

---

## 3. Технические детали

### Валидация на фронтенде (Zod):

- **Email:** валидный формат.
- **Password:** минимум 8 символов.
- **Name:** минимум 2 символа.

### Обработка ошибок:

- Использовать `try/catch` в компонентах форм.
- Выводить сообщения об ошибках от API пользователю (Toast или текст под формой).

### Безопасность:

- Токены сохраняются в `localStorage` (реализовано в `SessionManager`).
- API клиент автоматически добавляет заголовок `Authorization: Bearer <token>` (проверить в `shared/api/client.ts`).
